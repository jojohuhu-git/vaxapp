## What was built in this session (PRs #12–#17)

### PR #12 — Core UI improvements
- **Date mask in QuickAdd**: digits-only input auto-inserts slashes (MM/DD/YYYY). Backspace at position 3 or 6 skips the slash and removes the preceding digit.
- **Enter-to-submit QuickAdd**: pressing Enter from any field inside the card calls `handleAdd()`. Inline red error appears below the date field on validation failure.
- **Doses Recorded read-only**: removed the edit mode, brand dropdown, and "+ Dose" column from `HistoryTable.jsx`. Doses are now pill-only.
- **Responsive layout**: `overflow-x:visible` on the history table, `table-layout:fixed`, first column width clamped.

### PR #13 — Duplicate dose on Enter fix
- `handleDateKeyDown` was calling `handleAdd()` and then the event bubbled to the container's `onKeyDown` — triggering a second add. Fixed with `e.stopPropagation()` before `handleAdd()` in the date field's keydown handler.

### PR #14 — MenACWY overdose detection
- **Audit** (`validation.js`): `auditAll(hist, dob, risks)` now accepts `risks` as a third parameter. Added a `series_over` warning when a non-high-risk patient has ≥3 MenACWY doses.
- **Pill UI** (`HistoryTable.jsx`): computes `isHighRiskMen` from `state.risks`; passes `isExtra={vk === "MenACWY" && !isHighRiskMen && i >= 2}` to `DosePill`, turning the 3rd+ pill orange (`p-grace`).
- High-risk MenACWY criteria: `["asplenia","complement","complement_inhibitor","hiv"]`.

### PR #15 — UI flatten / style consistency
- Removed dose pill icons (status icons stripped from DosePill).
- Global border-radius vars: `--rad: 3px`, `--rads: 2px` (was 10px/6px) — rectangles everywhere.
- Left sidebar grid column widened: `300px → 396px`, `max-width: 1280px → 1380px`.
- Forecast font sizes unified at 10px (was 7.5–8.5px).
- Optimal Schedule chips: `humanLabel()` function maps raw constraint labels (e.g. `hepA.i[1]=182d`) to plain English (e.g. `"HepA: D2 ≥6 months after D1"`).

### PR #16 — Td vaccine + audit grouping + sidebar width
- **Td vaccine added**: `vaccineData.js` (VAX_META + VAX_KEYS + VBR), `scheduleRules.js`, `refs.js`, `recommendations.js` (`td` count, `totalTetanus = dt + tdap + td`).
- **Audit panel grouped by vaccine** (`AuditPanel.jsx`): errors are now consolidated — one card per vaccine, sub-entries separated by a hairline rule. Title strips the vaccine name prefix to avoid duplication.
- **`auditAll` call sites** updated in `AuditPanel.jsx` and `RecTab.jsx` to pass `state.risks`.

### PR #17 — PatientInfo age completeness + DOB mismatch alert
- **Age dropdown**: replaced sparse hardcoded list with dynamically generated `AGE_OPTIONS`:
  - Every month 0–23 (month 12 labeled "12 months (1 year)")
  - Every year 2–18, with a 4.5y (54m) entry between 4y and 5y
  - 19y–25y yearly
  - 30, 35, 40, 45, 50y for adult HPV shared-decision and other adult recs
- **DOB → age mismatch warning**: `dobToMonths(dob)` computes current age in whole months. If it diverges from the selected age beyond a tolerance, an inline red warning appears below the dropdown.
  - Tolerance: 1m (<24m), 3m (24–72m), 6m (72–144m), 12m (≥144m)
  - Warning text: "DOB suggests age X — does not match selected age."
- DOB field already had date masking from PR #12; this PR added the same pattern to the DOB field in `PatientInfo.jsx` (`applyDateMask` + `onKeyDown` backspace handler).

## Key invariants to preserve

1. **Never emit `r("DTaP", ...)` for patients ≥7y (84m+)** — always `r("Tdap", ...)`.
2. **`totalTetanus = dt + tdap + td`** in `recommendations.js` — Td counts toward the tetanus series.
3. **`auditAll` signature**: `auditAll(hist, dob, risks = [])` — risks required for MenACWY series-over check.
4. **MenACWY high-risk criteria** (audit + pill): `["asplenia","complement","complement_inhibitor","hiv"]`.
5. **`propagateMaxM` must NOT be on Pediarix/Vaxelis/Pentacel** — dose-number gates in `comboValidForDose` already enforce per-dose limits.
6. **Five-surface rule**: any rec logic change must be verified in all five output surfaces (recommendations, regimens, forecast, catch-up, optimal schedule).

## Editing recommendations.js

Always use Python — the file contains literal `\uXXXX` escape sequences that the Edit tool corrupts:

```python
with open('/absolute/path/to/src/logic/recommendations.js', 'r') as f:
    content = f.read()
content = content.replace(old, new, 1)
with open('/absolute/path/to/src/logic/recommendations.js', 'w') as f:
    f.write(content)
```

Use the **absolute worktree path** — relative paths have caused silent write failures.
